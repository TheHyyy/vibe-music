### 全流程排查与修复计划

1.  **启动开发环境**
    *   启动后端 (`npm run dev:server`) 和前端 (`npm run dev:web`) 服务。

2.  **MCP 全流程走查 (执行并记录问题)**
    *   **场景一：新建房间 (复现当前 Bug)**
        *   操作：点击“创建新房间”。
        *   检查：进房后的自动播放行为、Console 报错详情、Toast 提示内容。
    *   **场景二：点歌与播放**
        *   操作：使用搜索功能（如搜索“陈奕迅”），将歌曲加入队列。
        *   检查：歌曲是否成功上榜、是否自动开始播放、歌词是否加载、URL 获取是否正常。
    *   **场景三：播放控制**
        *   操作：点击暂停/播放、拖动进度条、点击“切歌”。
        *   检查：WebSocket 事件 (`player:update`) 是否发送、UI 状态是否同步。
    *   **场景四：加入房间**
        *   操作：复制房间码，刷新页面，使用“加入房间”功能重连。
        *   检查：是否能恢复之前的播放状态和队列。

3.  **修复策略**
    *   **解决核心 Bug (进房报错)**：根据场景一的日志，修复热歌推荐或自动播放的逻辑漏洞（如后端过滤无效热歌、前端优化错误提示）。
    *   **修复走查发现的其他问题**：如果在点歌、切歌等流程中发现异常（如歌词不同步、切歌卡顿），一并修复。

4.  **回归验证**
    *   再次执行上述关键流程，确保用户体验流畅，无明显报错。
